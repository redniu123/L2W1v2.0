## 4. 当前正在进行的步骤

### 阶段 E：正式拆出 BAUR-only，并与 SH-DA++ 拉开系统边界
**状态**：已完成（第一轮）

#### 本阶段目标
让系统级主实验 M4 / M5 成立：
- `BAUR-only`：只保留 BAUR 路由主链
- `SH-DA++`：BAUR + 领域软提示词 + 严格回填 + 熔断

#### 为什么必须做
现在虽然已经有：
- `BAUR`
- `DAR`
- `SH-DA++`

但还没有严格意义上的：
- `BAUR-only`（系统级对照方法）

如果不拆这个分支，就无法真正回答实验设计书中的：

> “完整 SH-DA++ 是否优于仅有 BAUR 的基础链路？”

#### 目标结果
脚本中应能正式跑出：
- `GCR`
- `BAUR`
- `DAR`
- `BAUR-only`
- `SH-DA++`

其中：
- `BAUR` / `DAR` 更偏 Router 对比
- `BAUR-only` / `SH-DA++` 更偏系统对比
