### 3.2 主实验 B：系统级方法
- 材料目录：`results/expriments/exB/`
- 输出规范：`docs/expB_output_spec.md`
- 整理模板：`results/expriments/exB/main_experiment_B_materials_template.md`

#### 当前冻结方向
- `M5 = BestRouter-only`
- `M6 = SH-DA++`
- BestRouter：当前固定为 `GCR`
- 选择依据：
  - `GCR` 在 `0.30` 预算取得全局最佳 CER
  - 更适合作为后续 `0.50 / 0.80` 扩展预算实验的统一固定 Router
  - 系统级实验后续统一沿用 GCR，避免跨预算叙事切换 Router

#### 当前重启背景
- 云服务器上的首轮主实验 B 在 `B=0.20` 过程中遇到断网风险。
- 当前原则：**主实验 B 需要按完整 run 重新启动**，不再沿用中断 run 进入正式材料。
- 重启后必须保证同一 `run_id` 下完整覆盖 `0.10 / 0.20 / 0.30` 三个预算点。

#### 主实验 B 重新启动任务
1. 在云服务器确认旧进程是否已退出，并记录异常原因。
2. 新建正式 run，重新执行 `M5 / M6` 的 `0.10 / 0.20 / 0.30` 在线预算实验。
3. 跑完后先检查 `budget_check.csv`，确认 `Actual_Call_Rate = B ± 0.5%`。
4. 再检查 `summary.csv`、逐 budget JSONL、`backfill_log.jsonl`、`breaker_log.jsonl` 是否齐全。
5. 若 run 不完整、预算点缺失、JSONL 缺失或网络异常未标注，则该 run 记为 `invalid`，不得进入论文主表。
