## 1. 研究背景

### 1.1 任务定义
本项目任务为：给定单行文本图像与 OCR 初始结果，输出最终修正文本。
研究对象聚焦 geology、finance、medicine 三个专业领域，不设置 general 域。

### 1.2 当前问题
纯 OCR 在复杂手写、边界漏字、生僻字、低置信度和专业术语场景下容易出错，而高性能 VLM 的调用成本较高，因此需要预算感知的路由机制来决定哪些样本值得升级到 Agent B。
若缺乏约束式回填与熔断保护，VLM 还可能引入自由改写、幻觉生成、格式坍塌和过纠错等问题。

### 1.3 当前论文目标
本文要证明的不是“某一个学习式 Router 必然最优”，而是“在预算约束下，合理的路由设计 + 端云协同 + 约束式纠错链路”能够有效提升多领域中文手写 OCR 纠错质量。

## 2. 研究目标与核心假设

### 2.1 总目标
在固定预算约束下，通过合理的 Router 与 SH-DA++ 框架，提高中文手写 / OCR 场景中的最终识别质量，并控制调用率、时延与可靠性。

### 2.2 子目标
1. 比较 `GCR / WUR / DGCR / DWUR` 在相同预算下的表现差异。
2. 验证 `r_d` 是否能稳定提升全局置信度路线与固定规则路线。
3. 将 `BAUR / DAR` 作为局部不确定性路线的补充组，分析其失败或受限边界。
4. 保证所有正式实验都能以逐样本 JSONL 的形式长期保存，以支持后续错误分析、论文绘图与审稿回复。

### 2.3 核心假设
- H1：GCR 是强全局置信度基线。
- H2：WUR 有可能优于纯 GCR，因为它融合了平均置信度、最小置信度和局部置信度突变信息。
- H3：DGCR 与 DWUR 可以检验 `r_d` 是否具有稳定增益。
- H4：BAUR / DAR 的表现边界与其局部排序信号是否能形成完整纠错闭环密切相关。

### 2.4 成功判定标准
- 在正式预算点上，主指标应稳定优于 AgentA_Only baseline。
- 正式在线预算实验要求 `Actual Call Rate = B ± 0.5%`。
- 主表结果必须来自统一评测脚本、冻结数据划分和可追溯的结果文件。
- 每个主实验都必须同时具备 summary 文件与 per-sample JSONL 文件。

## 4. 方法设计

### 4.1 系统总览
整体系统命名为 **SH-DA++**。
系统结构包括：Agent A 负责刚性 OCR 基线，Router 负责预算感知选样，Agent B 负责语义纠错，Backfill / Circuit Breaker 负责抗幻觉、拒改与熔断保护。

### 4.2 Agent A
正式实验中，Agent A 固定为基础 OCR 模块，用于输出初始文本和原始置信度信号。

### 4.3 主线路由器定义

#### GCR：Global Confidence Router
- 定义：基于 OCR 输出的全局置信度构造风险分数。
- 作用：作为当前最简单、最稳的强基线。
- 直觉：整体越不可信，越优先升级。

#### WUR：Weighted Uncertainty Router
- 定义：基于全局置信度、最小字符置信度和局部置信度突变构造固定规则分数。
- 主版本公式：

\[
q_{\text{WUR-base}} = 0.5(1-\text{mean\_conf}) + 0.3(1-\text{min\_conf}) + 0.2 \cdot \text{drop}
\]

- 其中：
  - `mean_conf`：整句平均置信度；
  - `min_conf`：整句单字符最小置信度；
  - `drop`：相邻字符置信度差值的最大值。

#### WUR-gated：门控增强版 WUR
- 在 `WUR-base` 基础上加入两个简单门控：
  - 若 `min_conf < 0.35`，则 `q += 0.10`
  - 若 `drop > 0.20`，则 `q += 0.10`
- 当前正式设计中，`WUR` 默认指 `WUR-gated`，而 `WUR-base` 作为其消融版本保留。

#### DGCR：Domain-Augmented Global Confidence Router
- 定义：在 GCR 基础上加入领域风险项 `r_d`。
- 形式：

\[
q_{\text{DGCR}} = q_{\text{GCR}} + \eta \cdot r_d
\]

- 目标：验证 `r_d` 加到全局置信度路线后是否有稳定增益。

#### DWUR：Domain-Augmented Weighted Uncertainty Router
- 定义：在 WUR 基础上加入领域风险项 `r_d`。
- 形式：

\[
q_{\text{DWUR}} = q_{\text{WUR}} + \eta \cdot r_d
\]

- 目标：验证 `r_d` 加到固定规则强基线后是否仍能提供额外收益。

### 4.4 补充路线保留

#### BAUR：Budget-Aware Uncertainty Router
- 定义：基于局部不确定性信号的预算感知路由器。
- 当前角色：不再作为主推路由，而是作为局部不确定性路线的补充实验和负结果分析对象。

#### DAR：Domain-Augmented Router
- 定义：在 BAUR 基础上加入领域风险项 `r_d` 的增强版本。
- 当前角色：用于验证 `r_d` 在 BAUR 路线中的补救作用，并作为边界分析对象保留。

### 4.5 最终系统定义
**SH-DA++ = Router + 领域软提示词 + 严格回填 + 熔断机制。**
其中 Router 在当前主线实验中优先取 `GCR / WUR / DGCR / DWUR`，而 `BAUR / DAR` 作为补充路线使用。

### 4.6 运行链路
1. Agent A 输出 `T_A` 与置信度相关信号。
2. Router 计算风险分数与是否升级的决策。
3. 若升级，则调用 Agent B 进行受限纠错。
4. Backfill 对输出执行路径约束、回填和拒改。
5. Circuit Breaker 根据风险信号进行保护性降级。

## 5. 模型矩阵

### 5.1 主文模型
主文固定为 **2 个本地 VLM + 2 个云端 VLM**，不再额外扩模，以控制实验复杂度并保持论文主线聚焦。

| 类别 | 模型 | 角色 |
|---|---|---|
| Local  | Qwen3-VL-8B  | 本地主力模型 |
| Local  | InternVL3-8B  | 本地对照模型 |
| Cloud  | Gemini 3 Flash Preview  | 云端高吞吐模型 |
| Cloud  | Claude Sonnet 4.6  | 云端强能力模型 |

### 5.2 使用原则
主实验先固定一个代表性 Agent B 比较路由器，再固定最优 Router 比较 4 个模型，从而把“路由收益”和“模型能力差异”分开讲清楚。

### 5.3 环境说明
当前本地硬件环境为 2080Ti / 22GB VRAM，因此本地 VLM 更适合低并发、异步排队或离线批处理，而不适合作为高并发真并发方案宣传。

## 6. 预算协议

### 6.1 Budget 定义
本项目中的 Budget 统一定义为 **Max Call Rate**，即在给定数据集上，允许被路由到 VLM 的样本最大比例。
Budget 不是 token 上限、显存上限或单样本时延预算。

### 6.2 正式在线预算点
正式在线预算点固定为：
- `B = 0.10`
- `B = 0.20`
- `B = 0.30` 

### 6.3 在线预算控制协议
在线预算默认采用 Online Control，而不是只使用静态阈值。
正式实验要求 `Actual Call Rate = B ± 0.5%`，否则该点结果无效。

### 6.4 离线预算回放协议
为提高预算实验效率，并支持预算-性能曲线分析，本项目保留离线预算回放协议。
该协议允许先对目标集合执行一次 100% 全量升级推理与缓存，再基于已保存的逐样本结果离线重建多个预算点下的结果文件。

#### 6.4.1 目的
- 快速分析不同预算点下的质量曲线。
- 复用一次 full-call 结果，减少重复调用成本。
- 为错误分析、案例筛选、绘图和领域比较提供统一数据底座。

#### 6.4.2 回放步骤
1. 在固定模型、固定 Prompt、固定数据划分下，对目标集合执行一次 100% 调用并缓存结果。
2. 保存完整 `full_budget_results.jsonl`。
3. 按 `router_score` 从高到低排序。
4. 对预算点 `B ∈ {0.05, 0.10, 0.20, 0.30, 0.50, 1.00}`，取前 `B` 比例样本作为 upgrade 集合。
5. upgrade 集合使用缓存的升级后结果，非 upgrade 集合回退到 `ocr_text`。
6. 为每个预算点重新生成逐样本 JSONL 和 summary 文件。

#### 6.4.3 使用边界
离线预算回放适合用于趋势分析、案例分析和曲线绘制。
离线预算回放不能完全替代正式在线预算实验，因为真实 call rate 控制、窗口波动、吞吐、排队和熔断触发仍需通过独立在线运行来观测。

## 7. Prompt、回填与可靠性

### 7.1 Prompt 规范
受限 Prompt 是系统的正式组成部分，不同 Prompt 版本必须记录为 `prompt_version`，若文本发生变化则必须升级版本号。
结果进入论文主表时，必须能回溯到对应 Prompt 版本。

### 7.2 严格回填
系统默认启用严格回填与全局拒改机制，用于约束 VLM 输出，降低幻觉与不必要的自由改写。

### 7.3 熔断机制
系统通过 Circuit Breaker 监控拒改 / 异常信号，在风险升高时执行保护性降级，避免错误被系统性放大。

### 7.4 VLM 专属错误桶
结果分析中应单独统计 VLM 引入的错误类型，如 hallucination、format collapse、over-correction、false rejection 等，这些错误不应与普通 OCR 错误完全混在一起。

### 8.1 总原则
**本项目所有正式实验结果都必须保存，且在线与离线都一样必须保存。**

## 9. 评测设计

### 9.1 核心指标
正式实验统一输出以下指标，并由统一脚本计算。

| 指标 | 定义 |
|---|---|
| CER  | 最终文本与 GT 的字符错误率  |
| Boundary Deletion Recall@B  | 边界漏字样本中被升级召回的比例  |
| Substitution CER  | 对齐后只统计 substitution 的错误率  |
| CVR  | 拒改或违反约束的比例  |
| AER  | 升级样本中被真正接受编辑的比例  |
| call rate  | 实际 VLM 调用比例  |
| p95 latency  | 端到端延迟的 p95  |
| token usage  | 模型消耗量  |

### 9.2 评测要求
- 所有指标必须基于统一评测脚本计算。
- summary 文件与 per-sample JSONL 必须同时保存。
- 每个预算点都必须能独立回查案例与错误类型。

### 9.3 输出文件
每轮正式实验至少输出：
- `per_sample_results.jsonl`
- `metrics_summary.json`

- `domain_breakdown.csv`

## 11. 执行顺序

### 11.1 冻结数据
先完成 `labels.jsonl` 检查、正式 split 固定和数据统计导出。

### 11.2 冻结主配置
再固定：
- Agent A
- 代表性 Agent B
- 主文模型矩阵
- Prompt 版本
- 正式 Budget 点
- WUR 的最终主版本（当前默认为 WUR-gated）

### 11.3 先跑主线，再跑补充路线
先完成 `M1-M6`，再进入 `V1-V4` 与 `S1-S2`。
这样可以先回答“当前最稳的路由主线是什么”，再讨论局部不确定性路线为何受限。

### 11.4 同时保存在线与离线结果
所有在线预算实验在正式运行时必须落盘 JSONL。
所有离线预算回放也必须为每个预算点单独落盘 JSONL。
在线与离线结果都要保留，后续分析时不允许只看其中一类。

## 12. 风险与应对

| 风险 | 表现 | 应对 |
|---|---|---|
| WUR 未优于 GCR | 固定规则融合收益不明显 | 回查权重与门控阈值，仅在 val 上调参  |
| DGCR / DWUR 无增益 | `r_d` 收益不稳定 | 分领域检查 geology / finance / medicine 的实际贡献  |
| BAUR 持续负收益 | 局部不确定性路线受限 | 保留为负结果分析，不再作为主推方法  |
| 云端成本高 | 无法横扫过多模型 | 保持主文 2 local + 2 cloud  |
| 本地吞吐不足 | 高并发不可行 | 本地走低并发或离线批处理  |
| 后续无法做案例分析 | 只有汇总数值，没有逐样本输出 | 强制保存全部 JSONL  |

## 13. 归档规范

### 13.1 版本命名
- 数据版本：`data_vYYYYMMDD` 
- 路由版本：`router_vX.Y` 
- 模型版本：`model_vX.Y` 
- Prompt 版本：`prompt_vX.Y` 
- 实验编号：`M / V / S / A + 两位数字` 
- 结果目录：`results/YYYYMMDD_runXX/` 

### 13.2 可追溯性
进入论文主表、主图和主要结论的所有结果都必须回溯到具体 `run_id`、数据版本、模型版本、路由版本和 Prompt 版本。

### 13.3 结果有效性判定
以下结果一律视为无效：
- 未记录配置的结果；
- 未保存逐样本 JSONL 的结果；
- 预算口径不一致的结果；
- 服务异常但未单独标注的结果；
- 临时改 Prompt 却未升级版本号的结果。

## 14. 当前冻结结论

当前项目的正式研究叙事如下：
- `GCR` 是当前最稳的全局置信度基线。
- `WUR` 是当前新的固定规则强基线。
- `DGCR / DWUR` 用于验证 `r_d` 是否具有稳定增益。
- `BAUR / DAR` 不删除，但转为补充实验、负结果分析与方法边界讨论对象。
- 主文模型矩阵固定为 Qwen3-VL-8B、InternVL3-8B、Gemini 3 Flash Preview、Claude Sonnet 4.6。
- 正式在线预算点固定为 10%、20%、30%。
- 离线预算回放用于趋势分析与案例回查，但不能替代正式在线预算实验。
- 在线结果和离线结果都必须完整保存，且每个预算点都必须生成单独 JSONL 文件。
